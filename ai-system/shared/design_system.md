# HSDNM Design System: Institutional Sovereignty v2.5

This document is the **Absolute Source of Truth** for all UI/UX development within the HSDNM Monorepo. All code generation MUST adhere to these rules.

## 1. The 60-30-10 Architecture
Every screen must be balanced according to these ratios:

### 60% — Foundation & Surface
- **Variable**: `--background` / `--surface`
- **Tones**: 
  - `Surface (White #FFFFFF)`: used for high-glare interaction areas.
  - `Foundation (Stone #F8F9FA)`: used as the global page background to eliminate screen fatigue.
- **Rule**: Avoid large blocks of pure white unless contained within a branded card or modal.

### 30% — Brand Presence (Authority)
- **Variable**: `--primary` / `--brand` / `--secondary`
- **Colors**:
  - `Deep Green (#004d3d)`: Primary navigation, core hulls.
  - `Forest Black (#001a14)`: Dark mode base, deep headers.
- **Rule**: Brand presence should feel institutional and "heavy." Use deep greens for all significant structural elements.

### 10% — Institutional Accent (Direction)
- **Variable**: `--accent` / `--gold`
- **Colors**:
  - `HSD Gold (#c9a227)`: **ONLY** for high-priority CTAs and "Gold Rails."
- **Rule**: Never use gold for body text. Gold is for action and structural separation (e.g., header borders).

## 2. Typography Strategy (Editorial vs Interface)
We use a bifurcated type system to indicate "Authority" vs "Function."

- **EDITORIAL (Serif)**: `Playfair Display`.
  - **Usage**: All H1, H2, large stats, and thematic quotes.
  - **Feel**: Prestigious, established, official.
- **INTERFACE (Sans)**: `DM Sans`.
  - **Usage**: Body copy, form labels, small buttons, metadata, technical dashboard data.
  - **Feel**: Modern, neutral, high-legibility.

## 3. Authoritative Morphology
- **Radius**: System base is **4px** (`rounded-md`). Institutional components should never feels "bubble-like."
- **Borders**: Standard border is **1px** (`--border`). The "Institutional Rail" (Header bottom) is **3px Gold**.
- **Shadows**: Use `shadow-sm` for cards. Avoid heavy blurs; use precise, short-distance shadows to imply paper-like depth.

## 4. Prohibited Patterns
- **No Pure Red**: Use HSD Gold or an "Alert Orange" if critical, but never default "Error Red."
- **No Soft Shapes**: No `rounded-2xl` or `rounded-full` on primary buttons.
- **No System Gradients**: Only use the official `bg-hero-gradient` (Forest-to-Deep-Green).
